Ampera Launch Plan v3 (Parallel Agent Teams, Skill-Driven)
Summary
This revision is optimized for parallel execution by multiple coding agents and applies these skills directly: vercel-react-best-practices, payload, vercel-composition-patterns, next-best-practices, next-cache-components.

Primary launch goal: full landing page + SEO + conversion forms (byggestrøm, LFS, contact) + end-to-end persistence + dashboard visibility, deployed fast on hybrid stack (Vercel frontend/Payload + AWS backend/Postgres/S3).

Assumptions and Defaults
Deployment remains hybrid: Vercel for frontend/Payload, AWS for backend/Postgres/S3.
Security credential rotation is a hard gate before production cutover.
Payload remains editable source for content/SEO/navigation; code remains source for design system and business logic.
Cache Components are enabled only after correctness is validated on staging.
Release target is fastest safe path, not full enterprise feature completion.
